# 传神智谱 PRD

版本: v0.1
日期: 2026-06-05
首个场景: 合同审核

## 1. 产品定位

传神智谱是企业经营决策分析平台的语义底座模块。它负责把企业业务中的对象、关系、行为、规则、场景和评测用统一结构管理起来，并发布给下游的传神智脑 Agent 平台调用。

传神智谱不定位为面向知识工程师的复杂本体建模工具，而是面向业务、法务、风控、经营分析和解决方案团队的低门槛语义资产管理平台。复杂的图谱存储、检索、版本、追溯和 Agent 装载逻辑放在后台，前台只暴露用户能理解的业务动作。

## 2. 背景与核心判断

合同审核场景中，企业真正需要沉淀的不是单个合同条款的问答能力，而是一套可持续演进的业务语义资产:

- 合同里有哪些业务对象: 合同、主体、标的、金额、付款计划、违约责任、交付义务、审批节点等。
- 对象之间有什么关系: 合同约束主体、付款计划对应标的、违约责任关联义务、审批节点作用于合同版本等。
- 哪些行为会发生: 提交审核、补充条款、触发审批、生成风险意见、要求人工复核等。
- 哪些规则应被 Agent 使用: 金额阈值、付款风险、主体资质、履约期限、管辖条款、违约责任不对等等。
- 哪些场景会激活哪些语义子集: 销售合同审核、采购合同审核、NDA 审核、服务合同审核、重大合同复核等。
- Agent 得出结论时如何追溯: 匹配了哪个场景、引用了哪些对象和规则、每一步推理怎样形成。

文章中的核心主张应直接体现在产品里: 本体负责定义，大模型负责推理。传神智谱要提供清晰、结构化、可评测、可追溯的业务语义定义，而不是把全部业务逻辑写成确定性规则引擎。

## 3. 产品目标

### 3.1 业务目标

1. 先完成合同审核场景的语义资产沉淀，让传神智脑可以基于统一本体执行合同风险识别、条款解释和审核建议生成。
2. 降低业务专家维护本体的门槛，让法务或业务管理员可以通过表单、模板和向导维护对象、关系、规则和场景。
3. 建立本体元素的版本、引用和影响范围追溯，便于复盘、审计和持续优化。
4. 后续再建立评测机制，用真实或模拟合同案例衡量本体和 Agent 的表现。

### 3.2 用户目标

业务管理员希望不用写 Cypher、OWL 或复杂代码，也能维护合同审核知识。

法务专家希望把合同审核经验拆成清晰规则，并看到规则是否被 Agent 正确使用。

Agent 开发者希望拿到稳定、版本化、按场景裁剪的语义包，而不是每次手工拼 Prompt。

企业管理者希望知道合同审核能力是否覆盖关键风险，以及最近版本是否可靠。

## 4. MVP 范围

当前版本只做本体管理，不做合同创建、合同录入、合同审核流转和合同生命周期管理。合同审核只是本体管理的首个业务域，用来定义对象、关系、行为、属性、场景和规则。

### 4.1 MVP 必须包含

- 本体空间初始化与管理: 使用“合同审核”模板生成基础本体资产，不创建合同业务单据。
- 对象类型管理: 合同、合同主体、条款、义务、付款计划、标的、风险事项、审批节点等。
- 关系类型管理: 包含、约束、关联、适用于、触发、依赖、冲突、替代等。
- 行为类型管理: 提交审核、识别风险、生成意见、请求补充材料、触发复核等。
- 属性字典管理: 属性、枚举、阈值、字段口径和引用范围。
- 规则管理: 条件、结果、场景标签、引用对象、引用关系、示例、严重级别、适用合同类型。
- 场景管理: 销售合同审核、采购合同审核、NDA 审核、服务合同审核。
- 本体版本管理: 草稿、待审核、已发布、已废弃，支持复制为新版本。
- 本体元素 CRUD: 所有管理型元素均支持新增、查看、编辑、删除/停用、复制、导入、导出。

### 4.2 MVP 暂不包含

- 合同创建、合同台账、合同录入、合同上传和合同在线编辑。
- 完整合同审阅工作流系统。
- 全量合同生命周期管理。
- 复杂审批流配置平台。
- 场景语义包正式发布到传神智脑。
- 评测集运行和推理轨迹查看。
- Agent 调用日志、密钥管理和线上调用监控。
- 面向所有经营决策场景的通用模板市场。
- 完整规则引擎或确定性推理机。
- 面向知识工程师的 OWL/SHACL 编辑器。

## 5. 核心概念

### 5.1 本体空间

一个业务领域的语义资产容器。MVP 中默认提供“合同审核”本体空间。

### 5.2 对象

业务中的核心实体类型。对象必须包含自然语言定义、关键属性、参与关系和示例。

合同审核对象示例:

- 合同
- 合同主体
- 条款
- 标的
- 付款计划
- 履约义务
- 违约责任
- 风险事项
- 审批节点
- 审核意见

### 5.3 关系

对象之间在特定场景下成立的连接。关系必须包含方向、定义、适用场景、成立条件和示例。

示例:

- 合同 包含 条款
- 条款 约束 合同主体
- 付款计划 对应 标的
- 违约责任 关联 履约义务
- 风险事项 触发 审批节点

### 5.4 行为

Agent 或业务系统可以执行或建议执行的动作模板。行为包含触发条件、输入参数、输出结果和副作用。

示例:

- 识别风险
- 生成审核意见
- 请求补充资质材料
- 触发重大合同复核
- 标记需人工确认

### 5.5 规则

面向大模型阅读和套用的原子业务规则。规则采用“条件 -> 结果”的形式，并带场景标签、引用对象、引用关系和已解示例。

规则不追求把所有复合判断写进一条规则，而是拆成可以由大模型按场景组装的原子规则。

### 5.6 场景

用于收敛上下文的业务语义切片。场景声明哪些对象、关系、行为和规则会被激活。

合同审核 MVP 场景:

- 销售合同审核
- 采购合同审核
- NDA 审核
- 服务合同审核
- 重大合同复核

### 5.7 评测集

用于衡量本体是否支撑 Agent 做出稳定结论的案例集合。每个评测案例包含输入事实、期望结论、期望引用规则和可接受偏差。

### 5.8 语义包

传神智谱发布给传神智脑的版本化产物。语义包按场景裁剪，包含对象、关系、行为、规则、示例和输出约束。

## 7. 关键用户流程

### 7.1 初始化合同审核本体

1. 用户选择“新建本体空间”。
2. 选择模板“合同审核”。
3. 系统生成默认对象、关系、场景和示例规则。
4. 用户补充企业自己的合同类型、金额阈值和风险等级。
5. 保存为草稿版本。

说明: 这里创建的是本体空间和语义资产，不创建合同业务单据。

### 7.2 维护一条合同规则

1. 用户进入“智谱 / 规则”，点击“新建规则”。
2. 选择场景，如“销售合同审核”。
3. 选择规则类型，如“付款风险”。
4. 用结构化表单填写条件和结果。
5. 填写自然语言解释和已解示例。
6. 系统自动检查引用对象、关系和属性是否存在。
7. 保存后进入草稿规则集。

### 7.3 查看与维护本体版本

1. 用户进入“智谱 / 版本记录”。
2. 查看本体对象、关系、行为、属性、场景和规则的变更。
3. 对比两个版本的差异。
4. 将草稿版本提交审核，或复制历史版本继续编辑。

### 7.4 发布给传神智脑

本流程为后续阶段，不进入当前 MVP。

1. 用户进入“智谱 / 语义包”。
2. 选择场景和版本。
3. 系统展示本次语义包包含的对象、关系、行为和规则。
4. 系统运行基础评测集。
5. 评测通过后，用户点击发布。
6. 传神智脑通过语义包 ID 或场景 ID 调用。

### 7.5 查看一次 Agent 审核追溯

本流程为后续阶段，不进入当前 MVP。

1. 用户打开某次合同审核记录。
2. 系统展示匹配场景、输入事实、激活规则、推理步骤和最终结论。
3. 用户可标记“结论正确”“规则漏用”“场景匹配错误”“事实抽取错误”。
4. 反馈进入评测集候选池。

## 8. 功能需求

### 8.0 智谱导航与功能清单

左侧一级导航只保留“智谱”。点击或展开“智谱”后展示二级功能。用户不需要在一级导航中看到“本体”“图谱”“Neo4j”等技术概念，所有能力都收敛为智谱下的业务功能。

智谱二级功能:

| 二级功能 | 功能定位 | CRUD 要求 | 阶段 |
| --- | --- | --- | --- |
| 工作台 | 查看本体资产状态和待办 | 查询、刷新、跳转处理 | 当前一期 |
| 对象 | 管理合同审核对象类型 | 新增、查看、编辑、删除/停用、复制、导入、导出 | 当前一期 |
| 关系 | 管理对象之间的业务关系 | 新增、查看、编辑、删除/停用、复制、导入、导出 | 当前一期 |
| 行为 | 管理 Agent 或业务系统可执行的动作模板 | 新增、查看、编辑、删除/停用、复制、导入、导出 | 当前一期 |
| 属性字典 | 管理对象属性、枚举、阈值字段 | 新增、查看、编辑、删除/停用、复制、导入、导出 | 当前一期 |
| 场景 | 管理合同审核语义切片 | 新增、查看、编辑、删除/停用、复制、预览 | 当前一期 |
| 规则 | 管理原子业务规则 | 新增、查看、编辑、删除/停用、复制、批量导入、批量导出 | 当前一期 |
| 版本记录 | 查看语义资产版本和变更差异 | 查询、查看、对比、复制为新版本、导出 | 当前一期 |
| 设置 | 管理空间、成员、权限、审批策略 | 新增、查看、编辑、删除/停用 | 当前一期 |
| 评测 | 管理场景评测案例和运行结果 | 新增、查看、编辑、删除/停用、复制、导入、导出、运行 | 后续阶段 |
| 推理轨迹 | 查看 Agent 调用后的业务审计链路 | 查询、查看、反馈、导出，不允许改写轨迹 | 后续阶段 |
| 语义包 | 管理发布给传神智脑的版本化产物 | 新增发布、查看、废弃、回滚、导出、复制为新版本 | 后续阶段 |
| 调用信息 | 管理传神智脑接入信息和调用日志 | 新增密钥、查看、编辑、删除/停用、刷新、导出 | 后续阶段 |

CRUD 统一约束:

- 已发布内容不允许物理删除，只能停用、废弃或复制为新版本。
- 删除或停用任何被引用元素前，必须展示影响范围。
- 所有新增、编辑、删除、停用、发布和回滚操作必须写入审计日志。
- 列表页默认支持搜索、筛选、排序、分页、批量选择和导出。

规则编写原则:

- 一条规则只表达一个最小判断。
- 条件必须能对应到对象、关系或属性。
- 结果必须能落到风险事项、审核意见、行为建议或标签。
- 每条规则至少有一个已解示例。

规则示例:

```yaml
规则名称: 高额预付款风险
场景标签: [采购合同审核]
风险类别: 付款风险
严重级别: 高
条件: 合同包含付款计划，且预付款比例大于企业采购合同默认阈值
结果: 标记风险事项为“高额预付款”，建议触发法务复核
引用对象: [合同, 付款计划, 风险事项]
引用关系: [合同包含付款计划]
示例: 采购合同总额 100 万，预付款 60 万，阈值 30% -> 标记高额预付款风险
```

- 查看评测案例

- 删除或停用评测案例

评测指标:

- 结论命中率
- 规则引用命中率

- 推理轨迹完整率
- 场景匹配准确率
- 输出结构合规率

后续评测门槛:

- 每个可发布场景至少 10 个评测案例。
- 发布前必须运行核心评测集。
- 关键场景评测通过率低于 90% 时不允许直接发布，需要负责人确认。

### 8.8 推理轨迹

后续阶段能力，不纳入当前一期 MVP。

一次 Agent 调用必须记录:

- 调用时间
- 调用方
- 语义包版本
- 匹配场景
- 输入事实
- 激活规则
- 规则套用顺序
- 中间结论
- 最终结论
- 结构化输出
- 用户反馈

轨迹展示目标不是暴露模型内部思维，而是展示业务可审计链路: 事实、规则、结论、证据和版本。

## 10. 数据与导入

MVP 支持三类输入:

- 手动创建: 通过表单创建对象、关系、规则和场景。
- 模板初始化: 使用内置合同审核模板生成初始语义资产。
- 批量导入: CSV 或 Excel 导入对象、关系和规则。

后续可支持从企业制度、合同模板、条款库和历史审核意见中半自动抽取候选对象和规则。

## 11. 非功能需求

### 11.1 易用性

- 普通业务用户不需要理解 Neo4j、Cypher、OWL 或 SHACL。
- 新建规则应在 3 分钟内完成。
- 从模板初始化合同审核本体空间应在 1 分钟内完成。
- 每个复杂页面都提供表格视图和局部图视图，不强制用户在大图中操作。

### 11.2 性能

- 常规列表查询响应时间小于 1 秒。
- 单次影响范围分析小于 10 秒。
- 批量导入 1000 条以内本体元素时，任务可异步执行并展示进度。

### 11.3 安全

- 支持角色权限控制。
- 支持发布审批。
- 记录所有关键变更。
- 本体空间、成员、角色和审计日志按租户隔离。

### 11.4 可追溯

- 已发布语义资产不可被覆盖修改。
- 每次新增、编辑、删除、停用、复制、导入和发布动作都必须记录操作者、时间和变更内容。
- 每个本体元素必须能追溯到版本、引用关系和影响范围。

## 12. 成功指标

MVP 成功指标:

- 业务用户可独立完成对象、关系、行为、属性、场景和规则的新增、查看、编辑、删除/停用。
- 新建一条规则草稿不超过 3 分钟。
- 停用被引用元素时，100% 展示影响范围。
- 所有本体元素变更 100% 写入审计日志。
- 批量导入对象、关系、属性和规则后，系统能展示成功、失败和重复数据明细。
- 当前一期不以合同审核准确率、Agent 调用稳定性或语义包发布为验收指标。

## 14. 主要风险与对策

| 风险 | 表现 | 对策 |
| --- | --- | --- |
| 产品过于概念化 | 用户不知道从哪里开始 | 默认进入合同审核工作台，提供模板、待办和下一步动作 |
| 本体维护门槛过高 | 业务用户依赖技术人员 | 用业务表单、规则向导和示例驱动编辑 |
| 规则粒度失控 | 规则太大或互相重叠 | 强制规则原子化，提供引用检查和重复提示 |
| 场景过多后难匹配 | Agent 激活错误语义包 | 建立场景定义、输入要求和评测集 |
| 大模型输出不稳定 | 同案不同判 | 低温度、结构化输出、评测门槛和轨迹复盘 |
| 图谱与真实业务漂移 | 本体长期无人维护 | 发布审批、反馈回流、版本差异和过期提醒 |

## 15. 待确认问题

1. 合同审核 MVP 优先覆盖哪些合同类型: 销售、采购、NDA、服务，是否需要加入劳动合同或租赁合同。
2. 规则阈值由平台统一维护，还是按企业、部门、合同类型分别维护。
3. 传神智脑调用语义包时，是否由传神智谱负责场景匹配，还是由传神智脑先匹配场景后再取包。
4. 是否允许在个别精确计算场景中引入外部函数，如金额比例、日期间隔、批量匹配。
5. 评测案例是否可以使用脱敏历史合同，还是先用人工构造案例。
