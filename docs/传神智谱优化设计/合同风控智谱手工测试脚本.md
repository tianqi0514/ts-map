# 合同风控智谱手工测试脚本

## 0. 初始化演示数据

后端运行在 `http://localhost:9010`，前端运行在 `http://localhost:9005/`。

```bash
python tools/seed_contract_risk_zhitu_demo.py --api-base http://localhost:9010
```

脚本会先删除所有现有本体空间，然后创建 `合同风控决策本体`，并自动完成函数测试、图谱查询、验证与发布。

初始化后应看到：

- 对象：10 个
- 对象属性：37 个
- 关系：10 个
- 函数与指标：4 个
- 行为：5 个
- 规则：6 个
- 查询能力：2 个

## 1. 本体列表

1. 打开 `http://localhost:9005/`。
2. 左侧一级导航保持在 `智谱`。
3. 页面应只展示一个本体：`合同风控决策本体`。
4. 点击本体卡片进入详情。

预期结果：

- 二级导航包含 `概览`、`对象`、`关系`、`函数与指标`、`行为与规则`、`查询能力`、`本体图谱`、`图遍历`、`验证与发布`、`版本记录`、`设置`。
- 不应再出现独立的 `属性字典`、`场景`、`规则` 导航。

## 2. 概览

1. 在 `概览` 查看对象、关系、函数、行为统计。
2. 点击快捷入口 `对象`、`关系`、`函数与指标`、`行为与规则`，确认能跳转。

预期结果：

- 统计数字与初始化数据接近。
- 快捷入口跳转正常。

## 3. 对象 CRUD 与对象属性

### 3.1 查看对象和属性

1. 进入 `对象`。
2. 点击 `合同`。
3. 在右侧详情中查看 `对象属性`。

预期结果：

- 节点只展示对象名，详情中展示属性。
- `合同` 下能看到 `合同编号`、`合同标题`、`合同类型`、`合同金额`、`币种`、`签署日期`、`到期日期`、`自动续约`。

### 3.2 新增对象

1. 点击 `新增对象`。
2. 填写：
   - 名称：`补充协议`
   - 编码：`Amendment`
   - 状态：`已启用`
   - 自然语言定义：`合同变更或补充约定。`
   - 结构化字段 JSON：

```json
{
  "display_name": "补充协议",
  "aliases": ["变更协议"],
  "business_owner": "法务风控部"
}
```

3. 保存。

预期结果：

- 对象列表出现 `补充协议`。

### 3.3 编辑、复制、停用对象

1. 对 `补充协议` 点击 `编辑`，把定义改为 `合同变更、补充或豁免约定。`。
2. 保存后点击 `复制`。
3. 对复制出的草稿对象点击 `停用`。

预期结果：

- 编辑后版本号增加。
- 复制对象名称带 `副本`。
- 停用对象状态变为 `已停用`。

### 3.4 新增对象属性

1. 打开 `合同` 详情。
2. 点击 `新增属性`。
3. 填写：
   - 属性名称：`违约金上限`
   - 中文展示：`违约金上限`
   - 编码：`Contract.penalty_cap`
   - 类型：`金额`
   - 必填：不勾选
   - 口径说明：`违约责任条款约定的最高赔偿金额。`
4. 保存。

预期结果：

- `合同` 的对象属性中出现 `违约金上限`。

## 4. 关系 CRUD

1. 进入 `关系`。
2. 查看初始化关系：`签约主体`、`包含条款`、`付款安排`、`产生义务`、`识别风险` 等。
3. 点击 `新增关系`。
4. 填写：
   - 名称：`存在补充协议`
   - 编码：`has_amendment`
   - 状态：`已启用`
   - 自然语言定义：`合同可能存在补充协议。`
   - 结构化字段 JSON：

```json
{
  "source_code": "Contract",
  "target_codes": ["Amendment"],
  "relation_kind": "business",
  "cardinality": "one_to_many",
  "business_meaning": "主合同与补充协议之间的变更关系。"
}
```

5. 保存后对该关系执行 `编辑`、`复制`、`停用`。

预期结果：

- 关系列表新增成功。
- 编辑、复制、停用状态变化正常。

## 5. 函数与指标

### 5.1 查看函数

进入 `函数与指标`，应看到：

- `高金额阈值判断`
- `付款计划金额求和`
- `最高风险分提取`
- `风险项计数`

### 5.2 新增函数

1. 点击 `新增函数与指标`。
2. 填写：
   - 名称：`付款逾期天数最大值`
   - 编码：`fn_max_overdue_days`
   - 状态：`已启用`
   - 自然语言定义：`提取付款计划中的最大逾期天数。`
   - 结构化字段 JSON：

```json
{
  "function_type": "max",
  "value_field": "overdue_days"
}
```

3. 保存。

预期结果：

- 函数列表新增成功。

## 6. 行为与规则

### 6.1 查看行为下的规则

1. 进入 `行为与规则`。
2. 点击 `升级法务复核`。
3. 在右侧详情中查看 `行为规则`。

预期结果：

- 规则在行为详情中展示，不再作为独立一级/二级功能出现。
- 能看到 `低信用主体触发法务复核`、`高风险分触发法务复核`、`自动续约缺少通知期触发复核`。

### 6.2 新增行为

1. 点击 `新增行为与规则`。
2. 填写：
   - 名称：`触发管理层审批`
   - 编码：`RequireManagementApproval`
   - 状态：`已启用`
   - 自然语言定义：`重大合同或阻断风险需要管理层审批。`
   - 结构化字段 JSON：

```json
{
  "action_type": "approval",
  "owner_role": "管理层",
  "sla": "1工作日"
}
```

3. 保存。

### 6.3 新增规则

1. 打开 `触发管理层审批` 详情。
2. 点击 `新增规则`。
3. 填写：
   - 规则名称：`阻断风险触发管理层审批`
   - 编码：`rule_block_risk_requires_management`
   - 类型：`硬规则`
   - 严重程度：`阻断`
   - 条件：`RiskFinding.severity == '阻断'`
   - 结果：`触发管理层审批并暂停签署。`
4. 保存。

预期结果：

- 新规则出现在该行为详情下。

## 7. 查询能力

1. 进入 `查询能力`。
2. 查看 `合同风控本体图谱查询` 和 `合同局部推理上下文查询`。
3. 新增查询能力：
   - 名称：`合同风险上下文查询`
   - 编码：`cap_contract_risk_context`
   - 状态：`已启用`
   - 自然语言定义：`读取合同相关风险、条款和审批上下文。`
   - 结构化字段 JSON：

```json
{
  "query_kind": "ontology_graph",
  "inputs": {
    "focus": "Contract"
  },
  "output_schema": {
    "nodes": "object[]",
    "edges": "relation[]"
  }
}
```

预期结果：

- 查询能力新增、编辑、复制、停用均正常。

## 8. 本体图谱

1. 进入 `本体图谱`。
2. 拖动对象节点，观察节点位置变化。
3. 点击 `合同` 节点。
4. 点击 `签约主体`、`包含条款`、`识别风险` 等关系边。

预期结果：

- 图谱只展示对象和关系。
- 对象节点只显示对象名称。
- 点击对象展示属性。
- 点击关系边展示该关系关联的规则或说明。
- 图谱为浅色模式，节点有轻微 3D 视觉效果。

## 9. 图遍历

1. 进入 `图遍历`。
2. 起点选择或输入 `合同` 对象。
3. 方向选择 `双向`。
4. 深度选择 `2`。
5. 执行遍历。

预期结果：

- 能看到合同到主体、条款、付款计划、风险项、审批记录的路径。

## 10. 验证与发布

1. 进入 `验证与发布`。
2. 点击 `运行验证`。
3. 检查校验项全部通过。
4. 点击 `发布本体`。

预期结果：

- 对象、关系、行为、函数/查询能力门禁均通过。
- 发布成功后可在版本记录中看到发布记录。

## 11. 版本记录

1. 进入 `版本记录`。
2. 查看初始化、编辑、验证、发布产生的记录。

预期结果：

- 能看到 `create`、`update`、`copy`、`run`、`publish` 等记录。

## 12. 设置

1. 进入 `设置`。
2. 查看本体名称、领域、编码、描述。

预期结果：

- 本体元信息与 `合同风控决策本体` 一致。

## 13. 建议测试顺序

建议按以下顺序测试，最容易发现数据流是否连贯：

1. 本体列表进入详情
2. 对象详情查看属性
3. 新增对象和对象属性
4. 新增关系
5. 新增函数
6. 新增行为和行为规则
7. 查看本体图谱
8. 图遍历
9. 运行验证
10. 发布本体
11. 查看版本记录
