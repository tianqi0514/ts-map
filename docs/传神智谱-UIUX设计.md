# 传神智谱 UI/UX 设计文档

版本: v0.1
日期: 2026-06-05
适用产品阶段: 合同审核 MVP

## 1. 设计目标

传神智谱的界面目标是让用户感觉自己在管理合同审核能力，而不是在操作一套复杂的知识图谱系统。

设计原则:

- 业务优先: 当前一期以本体管理为主线，先做好对象、关系、行为、属性、场景和规则的增删改查。
- 复杂后台化: Neo4j、图遍历、版本差异和语义包装载不作为主操作入口。
- 渐进展开: 默认展示列表、摘要、待办和关键指标，复杂关系通过抽屉、局部图和详情页展开。
- 可追溯但不压迫: 轨迹页面展示事实、规则、结论和版本，不暴露模型内部不可审计内容。
- 面向重复工作: B 端界面要紧凑、稳定、可筛选、可批量操作。

## 2. 目标用户与使用心智

### 2.1 法务专家

关心问题:

- 规则是否表达了真实审核经验。
- Agent 有没有用对规则。
- 某个风险结论为什么出现。

适合界面:

- 规则编辑器
- 评测
- 推理轨迹

### 2.2 业务管理员

关心问题:

- 本体空间是否完整。
- 哪些对象、关系和场景缺失。
- 当前本体版本有哪些待审核变更。

适合界面:

- 工作台
- 对象和关系管理
- 版本记录

### 2.3 Agent 开发者

关心问题:

- 应调用哪个语义包。
- 包里包含哪些对象和规则。
- 调用失败或结果异常如何排查。

适合界面:

- 语义包详情
- API 调用信息
- 调用日志

### 2.4 管理者和审计人员

关心问题:

- 审核能力是否可靠。
- 哪个版本产生了某次结论。
- 谁修改并发布了规则。

适合界面:

- 工作台概览
- 版本记录
- 审计日志

## 4. 全局布局

### 4.1 桌面端布局

- 左侧固定导航，宽度 220px，一级只显示“智谱”，二级功能在其下缩进展示。
- 顶部为当前空间、搜索、通知和用户入口，高度 56px。
- 主内容区采用列表 + 详情抽屉为主。
- 复杂详情使用右侧抽屉，宽度 520px 到 720px。
- 图谱视图只作为局部关系查看，不作为默认首页。

### 4.2 页面密度

适合 B 端中高密度布局:

- 列表行高 44px 到 52px。
- 表单分组清晰，避免大面积营销式卡片。
- 指标区域只放关键指标，不做装饰性大屏。
- 操作按钮固定在页面右上或抽屉底部。

### 4.3 视觉风格

- 配色采用传神语联官网视觉的红、蓝、紫、深灰组合。当前为官网视觉和页面 CSS 推定版，参考来源为传神语联官网，后续如拿到正式 VI 手册可直接替换色值。
- 品牌主色: 传神红 `#f8232e`，用于主按钮、当前导航、关键提醒、发布动作。
- 品牌强调色: 科技蓝 `#4c7df9`，用于链接、信息提示、局部图选中态、Agent 调用相关信息。
- 智能辅助色: 语联紫 `#c014e2`，用于 AI 推荐、智能抽取、规则建议等增强能力标识。
- 文本主色: 深灰 `#303030`，用于正文、标题和表格主信息。
- 页面背景: `#f5f7fa`，内容底色 `#ffffff`，分割线 `#e4e8ef`。
- 风险等级使用语义色: 高风险沿用传神红，中风险橙色 `#fa8c16`，低风险科技蓝，通过绿色 `#52c41a`。
- 页面背景浅灰，内容区域白底或弱边框。
- 卡片只用于单个信息单元，不做层层嵌套。
- 图谱节点颜色按类型区分，不用大面积渐变。

色彩使用约束:

- 传神红只用于最重要的行动和高风险提示，避免整页泛红。
- 科技蓝用于系统状态和可点击元素，保证 B 端界面稳定、清晰。
- 语联紫只作为智能能力点缀，不作为大面积背景。
- 左侧导航选中态使用浅红底 `rgba(248,35,46,.08)` + 传神红文字。

## 5. 关键流程

### 5.1 初始化本体空间

入口: 工作台空状态或顶部“新建空间”。

流程:

1. 选择模板: 合同审核。
2. 选择需要初始化的本体范围: 对象、关系、行为、属性、场景、规则。
3. 设置企业阈值: 重大合同金额、预付款比例、默认管辖地等。
4. 预览将生成的对象、关系、场景和规则。
5. 创建本体空间并进入工作台。

用户感知:

- 用户是在初始化合同审核领域的本体资产。
- 系统自动生成底层对象和关系。
- 用户只需要确认关键业务参数。
- 系统不创建合同业务单据，也不进入合同审核流程。

### 5.2 新增规则

入口: 智谱 / 规则右上角“新建规则”。

流程:

1. 选择场景和风险类别。
2. 填写条件。
3. 填写结果。
4. 选择引用对象、关系和属性。
5. 填写已解示例。
6. 系统做引用检查和可读性检查。
7. 保存草稿或提交审核。

规则编辑器采用分段表单:

- 基本信息
- 条件
- 结果
- 引用
- 示例
- 输出建议

关键交互:

- 条件和结果支持自然语言输入。
- 引用对象和关系用下拉选择。
- 当用户输入“付款比例”时，系统提示可引用“付款计划.预付款比例”。
- 当条件过长时，系统提示拆成多条规则。

### 5.3 编排场景

入口: 智谱 / 场景。

页面结构:

- 左侧场景列表。
- 中间为场景详情。
- 右侧为激活元素面板。

核心操作:

- 新增场景。
- 查看场景详情。
- 编辑场景定义。
- 删除或停用场景。
- 复制场景。
- 选择激活对象。
- 选择激活关系。
- 选择激活行为。
- 选择激活规则。
- 预览场景激活元素。

设计重点:

- 不让用户拖拽巨大图谱。
- 默认用分组清单管理激活元素。
- 提供“局部图预览”，只展示当前场景相关的一小片关系。

### 5.4 发布语义包

后续阶段流程，不纳入当前一期 MVP。

入口: 智谱 / 语义包，或场景详情页。

流程:

1. 选择场景和版本。
2. 查看发布前检查。
3. 运行或查看最近一次评测结果。
4. 填写变更说明。
5. 提交审核。
6. 审核通过后发布。

发布前检查样式:

- 通过项显示绿色对勾。
- 警告项显示黄色提示。
- 阻断项显示红色标记，并给出直接跳转链接。

### 5.5 查看推理轨迹

后续阶段流程，不纳入当前一期 MVP。

入口: 推理轨迹列表、Agent 调用日志或评测结果详情。

轨迹详情分区:

- 基本信息: 时间、调用方、语义包版本、场景。
- 输入事实: 合同片段或结构化事实。
- 激活规则: 本次被使用或候选的规则。
- 推理链路: 按步骤展示事实 -> 规则 -> 中间结论 -> 最终结论。
- 输出结果: 风险事项、严重级别、建议动作。
- 用户反馈: 正确、规则漏用、场景错误、事实错误。

注意:

- 不展示“模型思维链”字样。
- 展示的是业务审计链路和规则套用轨迹。

## 6. 核心页面设计

### 6.1 工作台

页面目标:

让用户一进来就知道合同审核语义资产是否可用、哪里需要处理。

主要模块:

- 顶部概览: 对象数、关系数、行为数、属性数、场景数、规则数。
- 待办列表: 草稿变更、规则待审核、引用异常、待处理删除/停用确认。
- 场景状态表: 场景、激活对象数、激活关系数、激活行为数、激活规则数、状态。
- 最近变更: 谁在什么时候新增、编辑、删除、停用了哪些本体元素。

主要操作:

- 新建对象
- 新建关系
- 新建规则
- 新建场景
- 查看引用异常
- 查看版本记录

### 6.2 智谱 - 对象

列表字段:

- 对象名称
- 编码
- 定义摘要
- 属性数
- 关系数
- 引用规则数
- 状态
- 最近更新

筛选:

- 状态
- 是否被规则引用
- 所属场景
- 更新时间

详情抽屉:

- 对象定义
- 属性
- 参与关系
- 关联规则
- 局部图
- 版本记录

空状态:

- 合同审核模板未初始化时，引导用户使用模板创建。
- 对象为空时，提供“从模板导入”和“手动新建”两个动作。

### 6.3 智谱 - 关系

列表字段:

- 关系名称
- 起点对象
- 终点对象
- 场景条件
- 是否可多跳
- 引用规则数
- 状态

关系详情:

- 自然语言定义
- 方向
- 成立条件
- 适用场景
- 示例
- 引用规则
- 局部图预览

### 6.4 智谱 - 行为

列表字段:

- 行为名称
- 输入参数
- 输出结果
- 关联场景
- 关联规则
- 状态

行为详情:

- 行为定义
- 前置条件
- 参数
- 结果
- 副作用
- 调用方

示例行为:

- 生成审核意见
- 请求补充材料
- 触发重大合同复核
- 标记人工确认

### 6.5 智谱 - 属性字典

页面目标:

统一管理对象属性、枚举值、阈值和字段口径，让规则引用同一套标准字段。

列表字段:

- 属性名称
- 属性编码
- 所属对象
- 数据类型
- 枚举值或取值范围
- 默认阈值
- 引用规则数
- 状态

主要操作:

- 新增属性
- 查看属性
- 编辑属性
- 删除或停用属性
- 复制属性
- 批量导入
- 导出

详情抽屉:

- 属性定义
- 取值约束
- 默认阈值
- 业务示例
- 引用规则
- 版本记录

### 6.6 智谱 - 规则

页面目标:

- 引用对象
- 引用关系

批量操作:

- 批量停用

- 批量导出

规则详情抽屉:

- 条件
- 结果
- 引用
- 示例
- 引用场景
- 变更记录
- 版本差异

### 6.7 规则编辑器

建议布局:

- 左侧为表单。
- 右侧为实时检查结果和引用预览。

表单字段:

- 规则名称
- 适用场景
- 风险类别
- 严重级别
- 条件
- 结果
- 引用对象
- 引用关系
- 已解示例
- 不适用示例
- 输出建议模板

实时检查:

- 对象是否存在
- 关系是否存在
- 条件是否过长
- 是否缺少示例
- 是否可能与已有规则重复

低门槛设计:

- 允许用户先写自然语言条件。
- 系统再推荐要绑定的对象和关系。
- 提交审核前强制通过完整检查。

- 错误率

主要操作:

- 新增调用密钥
- 查看调用信息
- 编辑权限范围
- 删除或停用密钥
- 刷新密钥
- 导出调用日志

### 6.14 智谱 - 设置

页面目标:

管理空间、成员、角色、审批策略和默认阈值。

主要操作:

- 新增成员、角色、审批策略
- 查看空间配置
- 编辑成员权限、发布策略、默认阈值
- 删除或停用成员、角色、策略
- 导出审计日志

## 7. 交互细节

### 7.1 搜索

全局搜索支持:

- 对象
- 关系
- 规则
- 场景
- 语义包
- 调用记录

搜索结果按类型分组，点击后进入详情抽屉或详情页。

### 7.2 详情抽屉

适用于对象、关系、规则、行为。

抽屉结构:

- 标题区: 名称、状态、版本、操作。
- 内容区: 定义、引用、示例、记录。
- 底部区: 保存、提交审核、关闭。

### 7.3 局部图

局部图只展示当前元素一跳或两跳关系。

交互:

- 点击节点查看摘要。
- 点击边查看关系定义。
- 支持切换一跳、两跳。
- 支持按场景过滤。

不在 MVP 中支持自由编辑大图，避免用户在图上迷路。

### 7.4 状态标签

状态:

- 草稿
- 待审核
- 已发布
- 已停用
- 有风险

风险等级:

- 高
- 中
- 低
- 提示

### 7.5 表单校验

即时校验:

- 必填字段为空。
- 编码重复。
- 引用不存在。
- 示例缺失。

发布前校验:

- 未绑定场景。
- 引用对象被停用。
- 缺少必填定义。
- 缺少示例。

### 7.6 反馈入口

所有 Agent 输出和评测结果都应提供反馈入口。

反馈类型:

- 结论正确
- 规则漏用
- 规则误用
- 场景匹配错误
- 事实抽取错误
- 建议不合适

反馈提交后进入“待处理反馈”，可转为评测案例或规则修改任务。

## 10. 可用性验收标准

1. 新用户能在 5 分钟内通过模板初始化合同审核本体空间。
2. 法务用户能在 3 分钟内新建一条完整规则草稿。
3. 用户能在 2 次点击内从工作台进入待处理事项。
4. 用户能从任一本体元素追溯到引用它的场景、规则和版本。
5. 用户在删除或停用被引用元素前能清楚看到影响范围和阻断项。
6. 对象、关系、规则列表均支持搜索、筛选和状态识别。

## 11. 后续设计问题

1. 场景数量增长后，是否需要提供场景分组和相似场景合并建议。
2. 规则自然语言编辑是否需要引入更强的智能辅助，如自动拆分原子规则。
3. 推理轨迹是否需要区分“规则候选”和“实际套用”。
4. 图谱视图是否应在后续支持只读全景图。
5. 企业制度、历史合同和审核意见的导入入口放在智谱二级功能，还是单独的数据源模块。
