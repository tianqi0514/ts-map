## 二、怎么跑（Neo4j，无需 LLM key）

**前置**：装一个 Neo4j（5.x，社区版即可，Docker 一行起）。

1. **灌库**：
   ```
   cypher-shell -u neo4j -p <你的密码> -f seed.cypher
   ```
   得到一张图：合同 / 主体 / 法人 / 股东 / 条款 / 政策 等节点与关系，含股权穿透边 `HAS_SHAREHOLDER`。
2. **跑三故事**：打开 `queries.cypher`，按注释分段在 Neo4j Browser 或 cypher-shell 里跑。
3. **对照基线（关键）**：另跑"纯规则视角"——`queries.cypher` 里**故事一 A3** 就是纯规则的查法（只查主体自己是否黑名单 → 返回 0 行 = 放行）。把它和 A2（穿透命中）并排，差异即图的价值。
4. **验收**：对照 `验证数据集.xlsx → 16_期望结果 golden`，逐条勾 A1–C3。

## 三、三故事怎么验（对应 queries.cypher）

- **故事一 · 图穿透（A1/A2/A3）**：`C-2024-007` 的新主体 S-301 自身全清白（A1），但 2~3 跳穿透发现其最终受益人王某同时控黑名单实体 S-900（A2）；纯规则只查 S-301 自身→放行（A3）。**这是 Neo4j 独有能力**，证图的价值。
- **故事三 · 本体一致性 + 可复现（C1/C2）**：C1 一条查询按"各合同适用的政策版本"算缺失强制条款，001/005/006 一起命中"缺数据合规"——改政策 `mandatoryClauses` 一处即全局一致；C2 对去年 C-2023-088 分别用 v1/v2 评，得"通过 / 缺条款"两种可溯结果。
- **故事二 · 多源取证（B1/B2，无需 LLM）**：用 `queries.cypher` 末段取主体图上事实，程序再并入 `seed.json` 的 `ExternalFact`（涉诉3起）+`CoopHistory`（2次延期），**用确定性模板归集**成一条定性提示 + 溯源（实缴50万<合同120万、成立<12月、涉诉3起、历史2次延期 → 关注履约、建议补担保）。B1/B2 可验；B3（自然语言问答/反幻觉）标"待大模型接入后验证"。
