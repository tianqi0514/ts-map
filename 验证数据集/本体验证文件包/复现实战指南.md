# 复现实战指南（闯关式）· 边搭边懂"本体 + 图 + manus 式 Agent"

> 给研发：**不是抄文档，是闯六关**。每关亲手搭起系统的一层，关里埋了"会失败的起点"和**核心 TODO 留白**——理解设计意图才能补对、才能通关。补全后用你自己的 LLM key 跑三个故事。
>
> 配套：`tool_backend.py`（8 工具 + 最小规则引擎 + 3 处 TODO）、`checks.py`（六关自测）、`tools.json`、`seed.json`、`seed.cypher`、`ontology.yaml`、`故事脚本_Agent版.md`、`运行说明_Agent.md`。

## 怎么用
1. 先 `python tool_backend.py` 看规则引擎自检能跑。
2. 逐关读"目标 / 你要搭什么 / 埋的难点 / 通关"，动手补 TODO 或实现。
3. 每关跑 `python checks.py N`（N=关号）看过没过；未过会告诉你卡在哪。
4. 六关全过后，按 `运行说明_Agent.md` 接你的 LLM key，跑三个故事的自主调查。

**起点状态**（未补 TODO 时跑 `python checks.py`）：关 3/4/6 已就绪，**关 1/2/5 故意未过**——这三处就是你要理解并补全的核心。

## 关 0 · 环境（搭"骨架"）
- **搭什么**：Neo4j（`docker run -p7687:7687 -e NEO4J_AUTH=neo4j/your_pwd neo4j:5`）；`pip install neo4j openai`；`cypher-shell -f seed.cypher` 灌图。设环境变量 `NEO4J_URI/USER/PWD`。
- **难点**：`tool_backend.py` 在连不上 Neo4j 时会**回退到内存图**（seed.json 的邻接表）——先用内存版通关也行，证明逻辑；再切 Neo4j 验证图库一致。
- **通关**：`python tool_backend.py` 打印出 rule_evaluate 结果。

## 关 1 · 本体落地（搭"世界模型与边界"）
- **目标**：让 Agent 取到本体切片，并且**越界字段读不到**。
- **搭什么**：`ontology_get_slice` 已给；你要补 `entity_get` 的**切片权限边界**。
- **埋的难点**：`【TODO-1】`。当前 `entity_get` 把所有字段都返回——这意味着"法务条款审核"切片也能读到 `creditScore`/`bankAcctTail`（越界！），故事二的"最小权限"直接崩。
- **怎么理解**：切片 = 声明式权限，不是建议。读取必须按切片过滤，越界进 `forbidden`。
- **通关**：`python checks.py 1` —— legal-procurement 切片下 `creditScore/bankAcctTail` 被挡进 forbidden。

## 关 2 · 图穿透（搭"关系工具"）
- **目标**：从 S-301 穿透到最终受益人王某。
- **搭什么**：补 `graph_traverse` 的**多跳 + 方向 + 防环**。
- **埋的难点**：`【TODO-2】`。当前只走 1 跳，只能到"海岳控股/陈某"，到不了王某（在第 2 跳）。
- **怎么理解**：关系推理是图的核心价值；穿透要 BFS/DFS、按 direction 取邻接、visited 防环。这步通了，故事一的"借壳"才查得动。
- **通关**：`python checks.py 2` —— S-301 三跳内到达"王某"。
- **进阶**：灌了 Neo4j 后，把内存遍历换成 Cypher `MATCH p=(s)-[:HAS_SHAREHOLDER*1..3]->(u) ...`，对比结果一致。

## 关 3 · 规则引擎（搭"确定性硬地基"）
- **目标**：同一合同按 v1/v2 政策评出**不同且可溯**的结论。
- **搭什么**：`rule_evaluate` 已给（最小实现，吃 `policyVersion`）。读懂它怎么按版本取强制条款清单。
- **埋的难点**：神经符号分工——**能写成规则的（缺条款、黑名单）归引擎，确定且可复现**；Agent 不重复做。
- **通关**：`python checks.py 3` —— C-2023-088 v1 通过 / v2 缺条款。
- **延伸**：正式落地时把这里替换成你们的规则引擎（Drools 等），契约不变。

## 关 4 · Agent 自主（搭"调查员"）
- **目标**：让模型**自己**决定深挖，查出 S-301 借壳——而不是你把步骤写死。
- **搭什么**：按 `运行说明_Agent.md` 的主循环接 LLM + `tools.json`；system prompt 用 manus 式（目标驱动、自主规划）。
- **埋的难点**：**别把调查流程写进代码**。S-301 自身字段全干净，只有当 Agent 自发"查 UBO→横向查王某还控谁→拼弱信号"才查得出。把步骤写死 = 退回 workflow = 故事失败。
- **通关**：`python checks.py 4` 先确认图能力就绪（弱信号可取、S-301↔S-900 有路径）；**A4 的最终判定**看 Agent 实际轨迹是否**未经预设自行发起**了横向拼图（对照 `故事脚本_Agent版.md` 故事一期望轨迹）。

## 关 5 · 语义 + 反幻觉（搭"读原文 + 证据校验"）
- **目标**：Agent 读条款原文挑出"责任上限被架空"，且**引用的证据 id 必须真实存在**。
- **搭什么**：`clause_read_text` 已给（返回原文）；补 `checks.py` 里的 `verify_citations` **反幻觉校验**。
- **埋的难点**：`【TODO-3】`。当前校验恒为 True——等于拦不住模型编造的证据 id。这是合规系统的命门。
- **怎么理解**：Agent 输出的每条结论都要引真实 id；校验器核对，编造的拦掉。
- **通关**：`python checks.py 5` —— 原文陷阱可读 + 真 id 通过 / 假 id 被拦。

## 关 6 · 融合判断（搭"差异化处置"）
- **目标**：对同样"缺数据合规"的 001/005/006，Agent 据上下文给出**不同**处置。
- **搭什么**：让 Agent 综合 `rule_evaluate` 结论 + 合同上下文（PII/配合度/进度）出处置策略并解释。
- **埋的难点**：这是**判断**不是查询——"缺条款"相同，但已付预付/涉PII/对方配合度不同，处置就该不同。统一动作 = 没体现 Agent 价值。
- **通关**：`python checks.py 6` 确认三份上下文各异；**C3 最终判定**看 Agent 是否真的给出三种不同处置且能解释取舍（对照故事三期望）。
